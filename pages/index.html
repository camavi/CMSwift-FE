Local Entry

`pages/` e un entrypoint locale del framework

Questo albero serve per demo locali, asset runtime compatibili e smoke test rapidi del framework. La documentazione narrativa non e parte di questo repository pubblico.

Dove lavorare adesso

Le sorgenti reali del framework sono in packages/core/src e packages/ui/src.

Apri demo UI Apri layout test Apri sample layout

Comandi corretti

  • npm run dev per avviare l'entrypoint locale sotto pages/
  • npm run build:runtime per rigenerare core, ui e bundle unificato
  • npm test per eseguire i test automatici del core

Cosa resta in `pages/`

  • asset runtime compatibili in pages/_cmswift-fe/
  • sample HTML locali e pagine di test rapide
  • nessun contenuto tutoriale editoriale come source of truth
